Introduction

What is exactly is CSS Flex Box?


The Flexbox Layout aims to provide an easier and more efficient way layout containers and items that respond to your needs. It is capable of doing so even when the size is unknown or dynamic.


The main purpose of the flex layout is to give the container the power to alter it's items heigh and width to fill the available space. The container can grow to fill available space, or shrink to prevent overflow.

Parent

The parent is the container, and in this case it is called the flex container. The parent has many properties that can help you alter how you want your items to respond in a flex environment.

display

This property defines a flex container. The container can be inline or block depending on the given value. This property is what enables flex context for all of the direct children.

flex-direction

This property sets the direction of the main axis. This determines the direction that the items are placed a flex container.

  • row (default): left to right
  • row-reverse: right to left
  • column: top to bottom
  • column-reverse: bottom to top

flex-wrap

By default all of the flex items will try to fit themselves onto one line. This can be changed with the flex-wrap property.

  • nowrap (default): all flex items will be on a single line
  • wrap: flex items will wrap on multiple lines going from top to bottom
  • wrap-reverse: flex items will wrap on multiple lines going from bottom to top

flex-flow

This is the shorthand method for the flex-direction and flex-wrap properties. Together these properties determine the flex container's main and cross axes.

justify-content

This property defines the alignment of the flex items along the main axis. If there is available extra space on a line when items are inflexible or have reached their maximum size.

  • flex-start (default): items are compiled at the start of the flex-direction
  • flex-end: items are compiled at the end of the flex-direction
  • start: items are compiled at the start of the writing-mode direction
  • end: items are compiled at the end of the writing-mode direction
  • left: items are compiled towards the left side of the flex-container. If that option does not work with flex-direction then it behaves like start
  • right: items are compiled toward the right side of the flex-container. If that option does not work with flex-direction then it behaves like start.
  • center: items are centered along the line
  • space-between: items are evenly distributed along the main axis.
  • space-around: items are distributed along the line with equal space around them. The spaces will not appear to be equal because the space between items combines the spaces of both.
  • space-evenly: items are spaced so the space between them is equal. The edges of container also have equal space between them and the first/last item

align-items

This property defines the layout of the flex items along the cross axis on the current line. The justify-content content equivalent for the cross-axis.

  • stretch (default): the flex items will stretch to fill the container, however min-width/max-width are respected
  • flex-start / start / self-start: items are placed at the start of the cross axis. The choice of value depends on respecting the rules of flex-direction or writing mode.
  • flex-end / end / self-end: items are placed at the end of the cross axis. The choice of value depends on respecting the rules of flex-direction or writing mode.
  • center: the items are centered along the cross axis
  • baseline: items are aligned so that their baselines(bottom of text) align.

align-content

If there is extra space along the cross axis, this property will determine that space in relation to the flex items. The justify-content equivalent for the cross axis.

  • stretch (default): lines stretch to take up available space
  • flex-start / start: items are compiled to the start of the container
  • flex-end / end: items are compiled to the end of the container.
  • center: items are centered in the container
  • space-between: items are evenly distributed along the cross axis.
  • space-around: items are distributed evenly with an equal amount of space between each line

Children

The children are the flex items that are placed within the parent or flex container. The flex-items have several properties that can be used to determine their behavior within the flex container.

order

By default flex items are laid out in the order of their source code. The order property can be used to alter the order in which they appear in the flex container.

flex-grow

This property defines the ability for a flex item to grow if necessary. The value given is a proportion that determines the amount of available space that a flex item should occupy.

flex-shrink

This property defines the ability for a flex item to shrink if necessary.

flex-basis

This property defines the default size of an item before the remaining space is distributed.

flex

This property is the shorthand for flex-grow, flex-shrink and flex basis. The shorthand is recommended as it sets the other values intelligently.

align-self

This allows the default alignment along the cross axis(or alignment specified by align-items property in the flex container) to be overridden for individual flex items.
